## 第1章 单片机应用基础概述

### 本章核心知识

- 冯·诺依曼结构：程序和数据使用统一存储空间，共用总线，CPU 顺序取指执行。
- 哈佛结构：程序存储器和数据存储器分开、分别寻址；51 单片机复习中经常结合程序存储器、数据存储器、`MOV/MOVC/MOVX` 区分来考。
- 单片机定义：把 CPU、存储器、I/O 接口、定时器/计数器、中断系统等集成在一个芯片上。
- 计算机系统组成：CPU、存储器、I/O 接口、总线、程序。
- 程序设计一般过程：分析任务、设计算法、编写程序、调试运行、整理资料。

### 必须掌握的概念

- **单片机**：面向控制的微型计算机芯片，集成 CPU、ROM/RAM、I/O、定时器、中断等资源。
- **总线**：地址总线、数据总线、控制总线。
- **RAM/ROM**：RAM 用于可读写数据，ROM/Flash 常用于程序存储。
- **机器语言、汇编语言、高级语言**：汇编语言是机器语言的符号化表示；C51 是 51 单片机常用高级语言。
- **结构化程序设计**：顺序、选择、循环三种基本结构。

### 重点概念详解

- **重点：冯·诺依曼结构与哈佛结构**
  - 冯·诺依曼结构把程序和数据放在同一存储空间。
  - 51 单片机复习中更重要的是“程序存储器”和“数据存储器”分开理解，因为后续指令 `MOVC`、`MOVX`、`MOV` 分别对应不同访问对象。

- **重点：单片机为什么适合控制**
  - 资料中强调单片机具有逻辑判断、定时计数、程序控制、中断处理、串行通信等功能。
  - 常用于智能仪表、PLC、家电、工业控制等场景。

### 常见考法

- 选择/填空：单片机由哪些基本部件组成。
- 简答：单片机与通用微机的区别。
- 判断：程序存储器和数据存储器是否统一编址。
- 联系后续章节考：用 `MOVC/MOVX/MOV` 区分访问空间。

### 易错点

- 不要把所有“存储器”混成一种：程序存储器、片内 RAM、片外 RAM、SFR 的访问方式不同。
- “汇编语言”不是高级语言，它是机器语言的符号化形式。
- 总线不是只有数据总线，系统扩展题中地址线、数据线、控制线都要画清楚。

### 自测小题

1. 单片机通常把哪些功能部件集成在一个芯片上？
2. 冯·诺依曼结构和哈佛结构的核心区别是什么？
3. 为什么单片机适合实时控制场景？

## 第2章 MCS-51 单片机硬件结构

### 本章核心知识

- 8051 内部组成：CPU、4KB ROM、低 128B 用户数据 RAM、4 个 8 位 I/O 口、1 个串行口、2 个 16 位定时/计数器、中断系统等；教材还把 80H～FFH 的 SFR 区纳入“256B 内部 RAM/SFR 空间”来说明。
- 片内数据存储器组织：工作寄存器区、位寻址区、用户 RAM 区、特殊功能寄存器 SFR。
- 4 组工作寄存器由 `PSW` 中 `RS1、RS0` 选择。
- `P0` 口常分时复用为低 8 位地址线和数据线；`P2` 口常作高 8 位地址线。
- 复位、时钟、最小系统、引脚功能是硬件基础。

### 必须掌握的概念

- **PC**：程序计数器，保存下一条要执行指令的地址。
- **SP**：堆栈指针，复位后常见考点是初值；资料中往年题出现“复位后 SP=07H”。
- **PSW**：程序状态字，含进位、辅助进位、溢出、奇偶、寄存器组选择等标志位。
- **SFR**：特殊功能寄存器区，用于控制 I/O、中断、定时器、串口等。
- **位寻址区**：51 的位操作能力是重点之一。

### 重点概念详解

- **重点：8051 基本资源**
  - 资料明确给出 8051 内部包含 4KB ROM、低 128B 用户数据 RAM、4 个 8 位 I/O 口、串行口、两个 16 位定时/计数器、中断系统；教材说明 80H～FFH 为 SFR 区。
  - 选择题和填空题常直接考数量。

- **重点：P0/P2 在系统扩展中的作用**
  - `P0`：分时作低 8 位地址线和 8 位数据线，扩展时需要 74LS373 锁存低 8 位地址。
  - `P2`：常提供高 8 位地址。
  - 模拟试卷选择题直接考“可分时作地址线和数据线的是 P0 口”。

- **重点：PSW 与工作寄存器组**
  - 4 组工作寄存器，每组 R0～R7。
  - 通过 `PSW` 中 `RS1、RS0` 选择当前工作寄存器组。
  - 中断或子程序中如果切换寄存器组或使用累加器，要注意保护现场。

### 常见考法

- 选择/填空：8051 有几个 I/O 口、几个定时器、几个中断源、片内 ROM/RAM 容量。
- 选择：P0/P2 在扩展外部存储器时的作用。
- 简答：PC、PSW、SP 的作用。
- 判断：SFR 能否当普通 RAM 用；PC 是否可直接寻址。
- 系统扩展：P0 接 74LS373 锁存低 8 位地址，P2 输出高位地址。

### 易错点

- `P0` 口作普通 I/O 时和作总线时含义不同；扩展外部总线时不要漏掉地址锁存。
- 不要把 `PC` 当作可直接寻址的 SFR。
- `SFR` 和片内低 128B RAM 不是同一块普通 RAM。
- `TI/RI` 是串口标志；`TF0/TF1` 是定时器溢出标志，软件清除要求不同，后面中断章会继续考。

### 自测小题

1. 8051 内部主要包括哪些功能部件？
2. P0 口在外部存储器扩展时承担什么作用？
3. `RS1 RS0 = 10` 时选中第几组工作寄存器？答：第 2 组，地址范围 10H～17H。

## 第3章 MCS-51 指令系统与汇编程序设计

### 本章核心知识

- 51 指令系统共有 111 条指令；资料给出单字节、双字节、三字节指令数量和机器周期信息。
- 指令格式：操作码 + 操作数。
- 常见操作数符号：`#data`、`direct`、`Rn`、`@Ri`、`DPTR`、`bit`、`rel`、`addr11`、`addr16`。
- 寻址方式：立即、直接、寄存器、寄存器间接、变址、位寻址、相对寻址等。
- 程序结构：顺序、分支、循环、子程序。
- 程序阅读题常考寄存器变化、循环次数、标志位、内存单元内容。

### 必须掌握的概念

- **立即寻址**：操作数本身在指令中，用 `#` 标记，如 `MOV A,#30H`。
- **直接寻址**：直接给出内部 RAM 或 SFR 地址。
- **寄存器寻址**：操作数在 `R0`～`R7` 等寄存器中。
- **寄存器间接寻址**：用 `@R0`、`@R1` 或 `@DPTR` 指向地址。
- **变址寻址**：常见于查表，如 `MOVC A,@A+DPTR`。
- **位寻址**：直接对可位寻址单元或 SFR 位操作。
- **相对转移**：`rel` 是补码形式的 8 位带符号偏移量。

### 重点概念详解

- **重点：`MOV/MOVC/MOVX` 区分**
  - `MOV`：常用于片内 RAM/SFR/寄存器之间的数据传送。
  - `MOVC`：访问程序存储器，常用于查表。
  - `MOVX`：访问片外数据存储器。
  - 往年题和 OCR 试卷多次出现该区分。

- **重点：程序阅读题方法**
  - 先看初始化：`R0/R1/R2/A/DPTR/SP` 初值。
  - 再看循环控制：`DJNZ`、`CJNE`、条件跳转。
  - 再看数据来源：`@R0/@R1` 是间接访问；`MOVX` 是片外数据；`MOVC` 是程序存储器查表。
  - 最后列出每次循环后的关键寄存器变化。

- **重点：堆栈与子程序**
  - `PUSH/POP` 用于保存/恢复现场。
  - `ACALL/LCALL` 调用子程序，`RET` 返回。
  - `RETI` 用于中断返回，不要和 `RET` 混用。

### 常见考法

- 选择：判断指令写法是否正确。
- 填空：写出寻址方式、指令执行后 A/寄存器/内存内容。
- 程序阅读：统计负数个数、数据块搬移、循环求和、查表。
- 程序设计：用汇编完成延时、端口读写、串口发送、存储器间数据传送。

### 易错点

- `#30H` 和 `30H` 不同：前者是立即数，后者可能是地址。
- `@R0/@R1` 是间接寻址；`R0/R1` 本身不是数据地址。
- `DJNZ` 先减 1 再判断是否为 0。
- `JNB ACC.7,...` 常用来判断最高位是否为 0；在有符号数中最高位可视作符号位。
- `RETI` 不是普通子程序返回。

### 自测小题

1. 说明 `MOV A,#30H`、`MOV A,30H`、`MOV A,@R0` 的区别。
2. `MOVC A,@A+DPTR` 通常用于访问哪类存储器？
3. 程序中 `DJNZ R0,LOOP` 的执行顺序是什么？

## 第4章 C51 程序设计及应用

### 本章核心知识

- C51 是 51 系列单片机常用 C 语言开发方式，资料强调其结构化、可读性强、库函数丰富、便于调试维护。
- C51 程序结构与标准 C 类似：函数、变量、预处理、先定义后使用。
- C51 与标准 C 的关键区别在于：51 有片内/片外程序存储器、片内/片外数据存储器、SFR、中断等特殊资源。
- 要掌握 C51 的数据类型、运算符、表达式、控制语句、数组、函数、指针。
- Keil 与 Proteus 联调是开发工具链内容。

### 必须掌握的概念

- **sfr/sbit**：用于定义特殊功能寄存器及位，资料提到标准 C 不直接定义 51 的 SFR。
- **控制语句**：`if`、`switch`、`while`、`do...while`、`for`。
- **数组/函数/指针**：程序设计题可能要求用 C51 实现端口读写或简单控制。
- **中断函数**：资料提到标准 C 没有单片机中断定义，C51 需要特殊写法；具体中断函数语法若题目未给，需按课堂/Keil 规则确认。

### 重点概念详解

- **重点：C51 与标准 C 的不同**
  - 51 单片机存储空间复杂，标准 C 不区分这些硬件空间。
  - C51 需要能访问 SFR、I/O、定时器、中断等硬件资源。

- **重点：C51 程序设计思路**
  - 先确定硬件资源：端口、定时器、串口、中断。
  - 再初始化寄存器。
  - 主循环中处理输入/输出。
  - 对周期性或突发事件使用定时器/中断。

### 常见考法

- 选择/填空：C51 与标准 C 的区别。
- 程序设计：端口输入输出、流水灯、按键扫描、串口发送。
- 读程序：判断 `if/for/while` 执行过程、变量变化。
- 简答：为什么 C51 适合单片机开发。

### 易错点

- 不要把普通 C 的内存模型直接套到 51；51 的程序存储器、片内 RAM、片外 RAM、SFR 有区别。
- C51 中访问硬件通常不是普通变量访问，需要通过寄存器/位定义。
- 程序设计题若指定“汇编为基础，可选 C51”，优先保证逻辑正确；C51 中断函数语法资料中给出为 `void 函数名(void) interrupt n [using m]`。

### 自测小题

1. C51 相比汇编语言的优势有哪些？
2. 为什么标准 C 不能直接描述 51 单片机的全部硬件资源？
3. 写 C51 程序控制端口前，为什么要先明确端口地址或 SFR 定义？

## 第5章 51 单片机中断系统

### 本章核心知识

- 中断：CPU 正在执行任务时，外部或内部事件请求 CPU 处理，CPU 暂停当前任务，转去执行中断服务程序，处理完再返回。
- 89C51 常见 5 个中断源：外部中断 0、定时器 0、外部中断 1、定时器 1、串行口。
- 中断控制相关寄存器：`TCON`、`SCON`、`IE`、`IP`。
- 中断响应、保护现场、恢复现场、中断返回是简答题重点。
- 中断服务程序入口地址固定，常在入口处放跳转指令。

### 必须掌握的概念

- **中断源**：产生中断请求的来源。
- **中断允许**：总允许位 `EA` 和各中断源允许位共同决定是否响应。
- **中断优先级**：高优先级可中断低优先级；同级按自然优先级查询。
- **中断向量地址**：各中断源服务程序固定入口。
- **保护现场/恢复现场**：保存并恢复程序运行所需寄存器、标志位等。

### 重点概念详解

- **重点：保护现场与恢复现场**
  - 资料题型说明直接出现简答：“子程序或中断服务程序中，为什么要保护现场？如何保护和恢复现场？”
  - 答题要点：进入子程序/中断后可能改变 A、PSW、工作寄存器、DPTR 等；用 `PUSH` 保存，用 `POP` 按相反顺序恢复。

- **重点：中断响应条件**
  - 有中断请求。
  - CPU 开中断，即 `EA=1`。
  - 对应中断源允许位为 1。
  - 无同级或更高级中断正在服务。
  - 当前指令周期结束；资料中 OCR 题提到若为 `RETI` 或访问 `IE/IP` 指令，还需紧接下一条指令执行完，关键细节建议人工确认。

- **重点：标志位清除**
  - 模拟试卷选择题强调“必须在中断服务程序中，用软件清除的中断请求标志是 TI、RI”。
  - 定时器溢出标志、外部中断标志是否硬件清除需按触发方式/资料细节区分，复习时不要一概而论。

### 常见考法

- 选择：哪些标志位需要软件清除；中断源个数；自然优先级顺序。
- 简答：中断概念、保护现场、响应条件。
- 程序设计：中断服务程序入口、`RETI`、`PUSH/POP`。
- 判断：中断向量地址能否任意设定；中断标志是否都由硬件清零。

### 易错点

- `RET` 和 `RETI` 不能混用。
- 保护现场时 `POP` 顺序要和 `PUSH` 相反。
- 串口中断由发送/接收共用，`TI`、`RI` 常需软件清除。
- 中断优先级和中断允许是两件事：允许决定能否响应，优先级决定多个请求时谁先处理。

### 自测小题

1. 为什么中断服务程序中要保护现场？
2. 89C51 的 5 个中断源分别是什么？
3. 串口中断中的 `TI` 和 `RI` 为什么容易成为考点？

## 第6章 定时器/计数器

### 本章核心知识

- 定时器/计数器本质是加 1 计数器。
- 定时方式：对内部固定频率的机器周期计数。
- 计数方式：对外部事件脉冲计数。
- 51 内部有两个 16 位定时器/计数器：`T0`、`T1`。
- 控制寄存器：`TMOD` 设置工作模式，`TCON` 控制启动/溢出标志。
- 4 种工作模式：模式 0、1、2、3；模式 1 和模式 2尤其常考。

### 必须掌握的概念

- **机器周期**：资料题型中考“晶振频率 2MHz 时，一个机器周期多少微秒”。51 常规 1 个机器周期 = 12 个振荡周期。
- **初值计算**：定时时间由机器周期、计数次数、溢出容量共同决定。
- **TR0/TR1**：启动控制位。
- **TF0/TF1**：溢出标志。
- **TMOD**：不能位寻址，设置时一次写入。

### 重点概念详解

- **重点：定时与计数区别**
  - 定时：计内部机器周期脉冲。
  - 计数：计外部输入脉冲，T0/T1 输入端与 P3.4/P3.5 有关。

- **重点：初值计算**
  - 资料中往年题出现 12MHz、6MHz、11.0592MHz 等晶振。
  - 基本思路：先算机器周期，再算需要计数次数，再用最大计数值减去计数次数得到初值。
  - 具体公式按工作方式位数确定；资料未明确所有模式公式时，需结合课堂例题核对。

- **重点：模式 2 自动重装**
  - 适合重复定时，也常用于串口波特率发生器。
  - 模拟试卷程序设计提示串口工作方式 2 波特率固定，不需计算波特率及设置 T1 初值，这是特定题目条件。

### 常见考法

- 选择/判断：定时器是否消耗 CPU 时间；TMOD 是否可位寻址；定时/计数工作原理。
- 填空：机器周期、初值、工作方式。
- 程序设计：定时中断、软件计数、延时、周期性输出。
- 串口题中关联：用定时器产生波特率。

### 易错点

- 晶振周期、机器周期、定时时间不要混淆。
- 模式 1 是 16 位，模式 2 是 8 位自动重装；不要混用初值范围。
- `TRx` 是启动位，`TFx` 是溢出标志。
- `TMOD` 不能位寻址。

### 自测小题

1. 定时器方式和计数器方式的输入脉冲来源有什么区别？

## 第7章 单片机串行口及应用

### 本章核心知识

- 串行通信：数据一位一位顺序传输，线少、适合远距离、速度相对低。
- 并行通信：多位同时传输，速度高、线多、适合短距离。
- 异步通信帧：起始位、数据位、可选校验位、停止位；资料中提到字符由 4 部分组成。
- 89C51 串行口有 4 种工作方式。
- `SBUF` 用于串口数据收发，发送和接收缓冲在物理上独立但地址相同。
- `SCON` 控制串行口，`TI/RI` 是发送/接收中断标志。

### 必须掌握的概念

- **波特率**：每秒传送二进制代码位数，单位 bit/s 或 b/s。
- **同步/异步通信**：资料中明确串行通信分同步和异步两类。
- **方式 0**：同步移位寄存器方式，可用于扩展 I/O。
- **方式 1/2/3**：具体帧格式和波特率设置需按课堂表格复习；资料中明确方式 1 和方式 3 波特率不是固定不变，OCR 题为判断题。
- **MAX232**：与 PC 通信时用于电平转换。

### 重点概念详解

- **重点：串口发送/接收流程**
  - 写 `SBUF` 触发发送。
  - 收到数据后从 `SBUF` 读。
  - `TI` 表示发送完成，`RI` 表示接收完成，常需要软件清零。

- **重点：串行口方式选择**
  - 方式 0 可作为同步移位寄存器扩展 I/O。
  - 多机通信常涉及方式 2/3；OCR 试卷中有“多机通讯工作方式”选择题，但 OCR 有误，具体答案建议人工核对。

- **重点：考试程序设计题**
  - 模拟试卷明确要求：每 20ms 读取外设 A 口，并采用串口工作方式 2 发送。
  - 提示说明：该题中串口工作方式 2 波特率固定，不需计算波特率及设置定时器 T1 初值。

### 常见考法

- 选择/判断：并行通信与串行通信特点、波特率定义、`SBUF`、`TI/RI`。
- 填空：异步通信帧格式。
- 程序设计：初始化串口、发送数据、轮询 `TI`、清 `TI`。
- 与中断结合：串行口中断服务程序。

### 易错点

- `SBUF` 读和写地址相同，但发送/接收缓冲物理上独立。
- `TI/RI` 不要等同于 `TF0/TF1`。
- 波特率不是“字节数/秒”，而是位数/秒。
- RS-232 与 TTL 电平不同，和 PC 通信需要电平转换。

## 第8章 单片机系统接口

### 本章核心知识

- 键盘接口：编码键盘、非编码键盘；独立式键盘、行列式键盘。
- 按键需要消抖，可用硬件或软件方法。
- LED 数码管显示：静态显示、动态显示。
- 接口设计常涉及 I/O 口方向、读引脚/读锁存器、限流电阻、扫描程序。
- 本章与程序设计题、简答题联系紧密。

### 必须掌握的概念

- **编码键盘**：由硬件译码器产生键值。
- **非编码键盘**：由软件扫描识别键值。
- **独立式键盘**：每个按键占用一根 I/O 线。
- **行列式键盘**：用行线和列线组合识别多个按键。
- **按键消抖**：避免一次按键被误读多次。
- **动态显示**：多位数码管轮流点亮，利用视觉暂留。

### 重点概念详解

- **重点：按键消抖**
  - OCR 试卷中有简答：“为什么要进行去抖动处理，如何处理。”
  - 答题要点：机械按键闭合/释放瞬间会抖动；软件可延时后再次检测，确认稳定后处理。

- **重点：读引脚与读锁存器**
  - 往年/OCR 资料多次出现“直接读端口”和“读端口锁存器”。
  - 读-改-写类操作针对锁存器，不等同于直接读引脚状态。

- **重点：LED 限流**
  - 题中可能要求说明为什么 LED 需要限流电阻，并用电源电压、导通压降、工作电流估算电阻。

### 常见考法

- 简答：按键为什么要消抖，如何消抖。
- 选择/判断：编码/非编码键盘区别；静态/动态显示区别。
- 程序设计：按键扫描、数码管动态显示。
- 计算：LED 限流电阻。

### 易错点

- 行列式键盘不是每个按键都独占一根线。
- 动态显示不是所有位一直同时点亮。
- 读端口引脚和读端口锁存器结果不一定相同。
- P0 口外接 LED 或作普通输出时，上拉问题要特别注意；资料中选择题和 OCR 题均出现该点。

## 第9章 单片机系统扩展

### 本章核心知识

- 最小系统：单片机正常运行所需的最简单配置，通常包括单片机、时钟电路、复位电路等。
- 系统扩展内容：程序存储器扩展、数据存储器扩展、I/O 端口扩展、A/D 和 D/A 接口。
- 51 可扩展 64KB 程序存储器和 64KB 数据存储器或 I/O 口。
- 外部总线：地址总线、数据总线、控制总线。
- 常用芯片：74LS373 锁存低 8 位地址；74LS245/244/273 等用于总线或 I/O 扩展；2732、6232 等存储器在模拟题中出现。

### 必须掌握的概念

- **地址锁存**：P0 分时输出低 8 位地址和数据，必须用 ALE 控制锁存器保存低 8 位地址。
- **程序存储器扩展**：通常与 `PSEN`、`EA`、地址线、数据线有关。
- **数据存储器扩展**：通常与 `RD`、`WR`、地址线、数据线有关。
- **片选**：由高位地址线或译码电路决定某个芯片是否被选中。
- **地址范围计算**：由芯片容量和片选地址线决定。

### 重点概念详解

- **重点：系统扩展题固定套路**
  - 看芯片容量，确定需要多少根地址线。例如 4KB 需要 12 根地址线。
  - 低 8 位地址来自 P0，经 74LS373 锁存。
  - 高位地址来自 P2。
  - 程序存储器接 `PSEN`；数据存储器接 `RD/WR`。
  - 按片选信号确定地址范围。

- **重点：2732 与 6232 模拟题**
  - 模拟试卷明确考：8051 扩展一片 EPROM 2732（4KB）和一片 SRAM 6232（4KB）。
  - 要求：补连接线、确定地址范围、编程把 2732 最低地址单元内容送到 6232 最高地址单元。
  - 具体地址范围依赖图中 P2.7、P2.6、P2.0～P2.3 等片选连接；资料图片中有图，但文字转写无法完整表达连线，**地址范围需人工结合原图确认**。

- **重点：I/O 端口扩展**
  - 模拟试卷程序设计题出现 74LS245 扩展输入口。
  - 要先确定外设 A 口地址，再编程周期性读取并串口发送。

### 常见考法

- 系统扩展题：补连线、片选、地址范围、用 `MOVC/MOVX` 传送数据。
- 选择：74LS373 锁存低 8 位地址。
- 程序设计：访问外部数据存储器或外部 I/O。
- 简答：最小系统与扩展系统区别。

### 易错点

- 4KB = 4096B，需要 12 根地址线，不是 13 根。
- P0 不能同时直接当稳定地址线和数据线，低 8 位地址必须先锁存。
- 程序存储器和数据存储器控制信号不同：`PSEN` 对程序存储器，`RD/WR` 对外部数据存储器。
- 地址范围不是只看容量，还要看片选连接。

## 第2章强化：MCS-51硬件结构

### 必背知识点

- **8051 典型资源**：4KB ROM、低 128B 用户数据 RAM、SFR 区、4 个 8 位 I/O 口、1 个串行口、2 个 16 位定时/计数器、中断系统。
- **P0**：外扩时分时复用为低 8 位地址和 8 位数据，必须用 74LS373 锁存低 8 位地址。
- **P2**：外扩时常输出高 8 位地址。
- **PSW**：保存 CY、AC、OV、P、RS1、RS0 等标志和工作寄存器组选择位。
- **SP**：堆栈指针，往年题常考复位初值和压栈/出栈。

### 考试常考问法

- “PSW 各位有什么作用？”
- “片内 RAM、位寻址区、SFR 如何区分？”
- “复位后 CPU 从哪里开始执行程序？”

### 易错点辨析

| 易错说法 | 正确辨析 |
|---|---|
| P0 是普通 I/O，外扩时也能直接稳定输出地址 | 错。P0 分时复用地址/数据，低 8 位地址必须锁存。 |
| P2 也要锁存低 8 位地址 | 错。P2 常提供高 8 位地址。 |
| PC 是 SFR，可以用普通指令读写 | 错。PC 不是普通可直接寻址 SFR。 |
| SFR 可以当普通 RAM 随便使用 | 错。SFR 控制硬件功能，不能当普通 RAM。 |

| I/O 口 | P0、P1、P2、P3，共 4 个 8 位口 |

| 串行口 | 1 个全双工串口 |
| 外扩程序空间 | 可扩展 64KB，具体以资料/芯片为准 |
| 外扩数据空间 | 可扩展 64KB，具体以资料/芯片为准 |

## 第3章强化：指令系统与汇编程序设计

### 必背知识点

- **111 条指令**：资料明确提到 51 指令系统共有 111 条。
- **`#data` 是立即数，`direct` 是直接地址，`@R0/@R1` 是寄存器间接寻址。**
- **`MOV` 访问片内数据/SFR，`MOVC` 查程序存储器，`MOVX` 访问片外数据存储器。**
- **`DJNZ`：先减 1，再判断是否为 0。**
- **`RET` 用于子程序返回，`RETI` 用于中断返回。**

### 考试常考问法

- “下列指令采用什么寻址方式？”
- “判断某条指令写法是否正确。”
- “程序执行后 A、R0、R1、R2、某 RAM 单元的值是多少？”
- “根据程序判断功能。”
- “写程序统计、搬移、查表、端口输入输出。”

### 易错点辨析

| 易错点 | 辨析 |
|---|---|
| `MOV A,#30H` 和 `MOV A,30H` 混淆 | 前者把立即数 30H 送 A；后者把地址 30H 单元内容送 A。 |
| `@R0` 当成 R0 的值 | `@R0` 表示以 R0 内容为地址，访问该地址中的数据。 |
| `MOVC` 与 `MOVX` 混用 | `MOVC` 查程序存储器；`MOVX` 访问片外数据存储器。 |
| `DJNZ` 循环次数算错 | 先减 1，再判断是否跳转。 |

### 速背表：常见寻址方式

| 写法 | 含义 | 例子 |
|---|---|---|
| `#data` | 立即寻址 | `MOV A,#55H` |
| `direct` | 直接寻址 | `MOV A,30H` |
| `Rn` | 寄存器寻址 | `MOV A,R0` |
| `@Ri` | 寄存器间接寻址 | `MOV A,@R1` |
| `@A+DPTR` | 变址寻址/查表 | `MOVC A,@A+DPTR` |
| `bit` | 位寻址 | `SETB P1.0` |
| `rel` | 相对转移 | `SJMP LOOP` |

### 典型程序案例1：统计 30H～34H 中负数个数

**题型来源倾向**：模拟题第一页程序阅读题出现类似程序。

#### 汇编写法

```asm
        ORG 2000H
START:  MOV R0,#05H      ; 共处理 5 个数据
        MOV R1,#30H      ; 数据起始地址 30H
        MOV R2,#00H      ; R2 用来统计负数个数

LOOP:   MOV A,@R1        ; 取当前数据
        JNB ACC.7,NEXT   ; ACC.7=0 表示最高位为 0，不是负数
        INC R2           ; 最高位为 1，按补码看作负数
NEXT:   INC R1           ; 指向下一个数据
        DJNZ R0,LOOP     ; 处理完 5 个数后结束
        SJMP $           ; 停在此处
        END
```

#### C51写法

```c
#include <reg51.h>

void main(void)
{
    unsigned char i;
    unsigned char count = 0;
    unsigned char idata *p = (unsigned char idata *)0x30;  // 指向片内 RAM 30H

    for (i = 0; i < 5; i++) {
        if (p[i] & 0x80) {     // 最高位为 1，按补码看作负数
            count++;
        }
    }

    while (1);                 // 程序停住，便于观察 count
}
```

#### 功能与知识点

- 功能：统计 5 个带符号数中负数的个数。
- 体现知识点：寄存器间接寻址、`ACC.7` 位判断、`DJNZ` 循环、补码符号位。
- 编程思路：确定数据区 -> 设置计数器 -> 循环取数 -> 判断最高位 -> 更新统计值。

## 第4章强化：C51程序设计

### 必背知识点

- **C51 与标准 C 的核心区别：C51 要直接面对 51 的 SFR、位、片内/片外存储器、中断和 I/O。**
- C51 程序由函数组成，变量和函数遵循先定义后使用。
- `sfr`、`sbit` 常用于定义特殊功能寄存器和位。
- 程序设计题要先写清硬件资源：哪个端口输入、哪个端口输出、是否用定时器/中断/串口。

### 考试常考问法

- “C51 相比汇编有什么优点？”
- “C51 与标准 C 的主要区别是什么？”
- “写一个端口输出/流水灯/按键控制程序。”
- “把汇编逻辑改写为 C51。”

### 易错点辨析

| 易错点 | 辨析 |
|---|---|
| 把 C51 当普通 PC C 语言 | 错。C51 需要访问 SFR、端口、中断、存储区。 |
| 端口变量未定义就直接用 | 应使用头文件或 `sfr/sbit` 定义。 |
| 延时函数精确当作定时器 | 软件延时受晶振和编译优化影响，精确定时应使用定时器。 |
| C51 中断语法随便写 | 具体语法依 Keil C51，资料未完整说明时需人工核对。 |

### 典型程序案例1：P1.0 LED 闪烁

#### 汇编写法

```asm
        ORG 0000H
START:  CPL P1.0          ; P1.0 取反，控制 LED 状态
        ACALL DELAY       ; 软件延时
        SJMP START        ; 循环执行

DELAY:  MOV R7,#200
D1:     MOV R6,#250
D2:     DJNZ R6,D2
        DJNZ R7,D1
        RET
        END
```

#### C51写法

```c
#include <reg51.h>

sbit LED = P1^0;

void delay(void)
{
    unsigned int i, j;
    for (i = 0; i < 200; i++) {
        for (j = 0; j < 250; j++);
    }
}

void main(void)
{
    while (1) {
        LED = !LED;       // LED 状态取反
        delay();          // 软件延时
    }
}
```

#### 功能与知识点

- 功能：让 P1.0 连接的 LED 周期性闪烁。
- 体现知识点：位操作、循环结构、子程序/函数、端口输出。
- 编程思路：初始化端口默认状态 -> 主循环中改变输出 -> 延时 -> 重复。

## 第5章强化：中断系统

### 必背知识点

- **5 个中断源**：外部中断0、定时器0、外部中断1、定时器1、串行口。
- **中断控制寄存器**：`IE` 控制允许，`IP` 控制优先级，`TCON/SCON` 存放部分请求标志。
- **中断返回必须用 `RETI`。**
- **保护现场：PUSH；恢复现场：POP，顺序相反。**
- **串口中断标志 `TI/RI` 常需软件清除。**

### 考试常考问法

- “什么是中断？”
- “为什么要保护现场？如何保护和恢复？”
- “89C51 有哪些中断源？”
- “中断响应需要满足哪些条件？”
- “哪些中断标志需要软件清除？”

### 易错点辨析

| 易错点 | 辨析 |
|---|---|
| 中断服务程序最后写 `RET` | 错。中断返回应使用 `RETI`。 |
| 保护现场只保存 A | 不够。还可能要保存 PSW、DPTR、工作寄存器等。 |
| 所有中断标志都硬件清零 | 错。串口 `TI/RI` 常需软件清零。 |
| 开了 EA 就一定响应中断 | 还要对应中断源允许位为 1，并满足优先级和当前指令条件。 |

### 中断速背表

| 内容 | 记忆点 |
|---|---|
| 总允许位 | `EA` |
| 外部中断0 | `INT0` |
| 定时器0 | `T0` |
| 外部中断1 | `INT1` |
| 定时器1 | `T1` |
| 串行口 | `TI/RI` 相关 |
| 中断返回 | `RETI` |

### 典型程序案例1：外部中断0触发后翻转LED

#### 汇编写法

```asm
        ORG 0000H
        LJMP MAIN

        ORG 0003H          ; 外部中断0入口地址
        LJMP INT0_ISR

MAIN:   SETB EA            ; 开总中断
        SETB EX0           ; 允许外部中断0
        SETB IT0           ; 边沿触发，具体触发方式按题目要求
        SJMP $             ; 主程序等待中断

INT0_ISR:
        PUSH ACC           ; 保护现场
        PUSH PSW
        CPL P1.0           ; 中断发生时翻转 LED
        POP PSW            ; 恢复现场，顺序与 PUSH 相反
        POP ACC
        RETI               ; 中断返回
        END
```

#### C51写法

```c
#include <reg51.h>

sbit LED = P1^0;

void int0_service(void) interrupt 0
{
    LED = !LED;            // 外部中断0触发后翻转 LED
}

void main(void)
{
    EA = 1;                // 开总中断
    EX0 = 1;               // 允许外部中断0
    IT0 = 1;               // 边沿触发

    while (1);             // 等待中断
}
```

#### 功能与知识点

- 功能：外部中断0到来时，P1.0 输出翻转。
- 体现知识点：中断允许、入口地址、保护现场、`RETI`、C51 中断函数。
- 编程思路：设置触发方式 -> 开中断 -> 主程序等待 -> ISR 完成短任务并返回。

## 第6章强化：定时器/计数器

### 必背知识点

- **定时器/计数器本质是加 1 计数器。**
- **定时方式**：计内部机器周期。
- **计数方式**：计外部脉冲。
- **`TMOD` 设置方式，`TCON` 控制启动和溢出。**
- **模式1是16位，模式2是8位自动重装。**
- **12MHz 晶振下，常规 51 的 1 个机器周期 = 1us。**

### 考试常考问法

- “定时器和计数器有什么区别？”
- “某晶振频率下机器周期是多少？”
- “定时器初值如何计算？”
- “TMOD/TCON 中相关位有什么作用？”
- “写一个定时中断程序。”

### 易错点辨析

| 易错点 | 辨析 |
|---|---|
| 晶振周期等于机器周期 | 错。常规 51 一个机器周期为 12 个振荡周期。 |
| 模式1和模式2初值算法相同 | 错。模式1 16位，模式2 8位自动重装。 |
| `TR0` 是溢出标志 | 错。`TR0` 是启动位，`TF0` 是溢出标志。 |
| 定时器工作一定占用 CPU 全程 | 定时器硬件计数，不需要 CPU 每个周期参与。 |

### 定时器速背表

| 项目 | 作用 |
|---|---|
| `TMOD` | 设置 T0/T1 工作方式和定时/计数选择 |
| `TCON` | 启动控制、溢出标志、外部中断控制 |
| `TR0/TR1` | 启动 T0/T1 |
| `TF0/TF1` | T0/T1 溢出标志 |
| 模式1 | 16 位定时/计数 |
| 模式2 | 8 位自动重装 |

### 典型程序案例1：T0定时中断实现1秒翻转LED

假设晶振为 12MHz，T0 模式1，每 50ms 中断一次，20 次约为 1s。50ms 初值常用 `3CB0H`。若课堂给出不同晶振或初值，以课堂为准。

#### 汇编写法

```asm
        ORG 0000H
        LJMP MAIN

        ORG 000BH          ; T0 中断入口
        LJMP T0_ISR

MAIN:   MOV SP,#60H
        MOV R7,#20         ; 20 次 50ms = 1s
        MOV TMOD,#01H      ; T0 模式1，16位定时
        MOV TH0,#3CH       ; 50ms 初值高字节
        MOV TL0,#0B0H      ; 50ms 初值低字节
        SETB ET0           ; 允许 T0 中断
        SETB EA            ; 开总中断
        SETB TR0           ; 启动 T0
        SJMP $             ; 主程序等待

T0_ISR: PUSH ACC
        PUSH PSW
        MOV TH0,#3CH       ; 重装初值
        MOV TL0,#0B0H
        DJNZ R7,EXIT       ; 未满 1s 则退出
        MOV R7,#20
        CPL P1.0           ; 1s 到，翻转 LED
EXIT:   POP PSW
        POP ACC
        RETI
        END
```

#### C51写法

```c
#include <reg51.h>

sbit LED = P1^0;
unsigned char tick = 0;

void timer0_service(void) interrupt 1
{
    TH0 = 0x3C;            // 重装 50ms 初值
    TL0 = 0xB0;

    tick++;
    if (tick >= 20) {      // 20 * 50ms = 1s
        tick = 0;
        LED = !LED;
    }
}

void main(void)
{
    TMOD = 0x01;           // T0 模式1
    TH0 = 0x3C;
    TL0 = 0xB0;
    ET0 = 1;
    EA = 1;
    TR0 = 1;

    while (1);
}
```

#### 功能与知识点

- 功能：用 T0 定时中断产生 1s 周期 LED 翻转。
- 体现知识点：模式1、初值重装、中断、软件计数。
- 编程思路：确定晶振 -> 计算单次定时 -> 设置初值 -> 中断中重装并累计次数。

## 第7章强化：串行口

### 必背知识点

- **串行通信线少、适合远距离；并行通信线多、速度高、适合短距离。**
- **波特率 = 每秒传输的二进制位数。**
- **`SBUF` 用于收发数据；发送和接收缓冲物理独立但地址相同。**
- **`TI` 发送完成标志，`RI` 接收完成标志，常需软件清零。**
- **方式0可作同步移位寄存器扩展 I/O；方式1/2/3 要结合帧格式和波特率复习。**

### 考试常考问法

- “串行通信与并行通信有什么区别？”
- “什么是波特率？”
- “`SBUF`、`TI`、`RI` 分别有什么作用？”
- “写串口发送程序。”
- “串口方式2有什么特点？”

### 易错点辨析

| 易错点 | 辨析 |
|---|---|
| 波特率等于字节/秒 | 错。波特率按 bit/s 计。 |
| 写 SBUF 和读 SBUF 是同一个物理缓冲 | 地址相同，但发送/接收缓冲物理独立。 |
| 发送完成后不用清 TI | 常需软件清 `TI`，否则下一次判断会出错。 |
| 串口方式都要定时器产生波特率 | 不一定。模拟题提示方式2波特率固定，不需设置 T1 初值。 |

### 串口速背表

| 符号 | 作用 |
|---|---|
| `SBUF` | 串口收发数据缓冲 |
| `SCON` | 串口控制寄存器 |
| `TI` | 发送完成标志 |
| `RI` | 接收完成标志 |
| `REN` | 允许接收 |
| `SM0/SM1` | 串口工作方式选择 |

### 典型程序案例1：读取P1口数据并用串口方式2发送

该例与模拟题“每 20ms 读取外设 A 口，并采用工作方式2发送”相似。这里用 P1 作为输入口演示；若题目给出外设 A 口地址，应把读取 P1 改为访问该外部 I/O 地址。

#### 汇编写法

```asm
        ORG 0000H
MAIN:   MOV SCON,#80H      ; 串口方式2，发送演示；是否 REN 按题目要求

LOOP:   MOV A,P1           ; 读取 P1 口数据
        MOV SBUF,A         ; 发送数据
WAIT:   JNB TI,WAIT        ; 等待发送完成
        CLR TI             ; 软件清发送标志
        ACALL DELAY20MS    ; 20ms 延时，题目可说明不用写出
        SJMP LOOP

DELAY20MS:
        ; 资料题型说明中提示 20ms 延时子程序可不用写出
        RET
        END
```

#### C51写法

```c
#include <reg51.h>

void delay20ms(void)
{
    // 题目若说明延时子程序不用写出，这里可留空或写软件延时
}

void main(void)
{
    SCON = 0x80;           // 串口方式2，发送演示

    while (1) {
        SBUF = P1;         // 读取 P1 并发送
        while (TI == 0);   // 等待发送完成
        TI = 0;            // 清发送标志
        delay20ms();
    }
}
```

#### 功能与知识点

- 功能：周期性读取并通过串口发送一个字节。
- 体现知识点：串口方式、`SBUF`、`TI`、端口读入、延时。
- 编程思路：设置串口方式 -> 读输入 -> 写 SBUF -> 等 TI -> 清 TI -> 延时循环。

## 第8章强化：I/O口、键盘与显示接口

### 必背知识点

- **P0 作普通 I/O 输出时常要注意上拉。**
- **读引脚和读锁存器不是一回事，读-改-写操作针对锁存器。**
- **键盘分编码/非编码，也可分独立式/行列式。**
- **按键必须消抖，否则一次按键可能被识别多次。**
- **LED 数码管显示分静态显示和动态显示。**

### 考试常考问法

- “为什么按键要消抖？如何消抖？”
- “编码键盘和非编码键盘有什么区别？”
- “独立式键盘和行列式键盘有什么区别？”
- “静态显示和动态显示有什么区别？”
- “为什么 LED 要接限流电阻？”

### 易错点辨析

| 易错点 | 辨析 |
|---|---|
| 按键按下一定只读到一次 | 错。机械抖动会造成多次误读。 |
| 行列式键盘每个键都占一根 I/O | 错。行列式用行线和列线组合识别。 |
| 动态显示就是一直点亮所有位 | 错。动态显示是快速轮流点亮。 |
| P0 和 P1 输出能力完全一样 | 不应简单等同；P0 上拉问题是常考点。 |

### I/O接口速背表

| 接口内容 | 复习重点 |
|---|---|
| 独立按键 | 简单，占 I/O 多 |
| 行列键盘 | 节省 I/O，需要扫描 |
| 按键消抖 | 延时后复测，或硬件消抖 |
| 静态显示 | 占 I/O 多，程序简单 |
| 动态显示 | 占 I/O 少，需要扫描刷新 |
| LED限流 | 防止电流过大烧坏 LED |

### 典型程序案例1：按键控制LED并软件消抖

假设按键接 P1.0，低电平表示按下；LED 接 P1.7。

#### 汇编写法

```asm
        ORG 0000H
MAIN:   SETB P1.0          ; P1.0 作输入，先写 1
        SETB P1.7          ; LED 初始灭/亮取决于电路

LOOP:   JB P1.0,LOOP       ; P1.0=1 表示未按下
        ACALL DELAY        ; 消抖延时
        JB P1.0,LOOP       ; 再次确认是否仍按下
        CPL P1.7           ; 确认按下后翻转 LED

WAITUP: JNB P1.0,WAITUP    ; 等待按键释放
        SJMP LOOP

DELAY:  MOV R7,#200
D1:     DJNZ R7,D1
        RET
        END
```

#### C51写法

```c
#include <reg51.h>

sbit KEY = P1^0;
sbit LED = P1^7;

void delay(void)
{
    unsigned int i;
    for (i = 0; i < 5000; i++);
}

void main(void)
{
    KEY = 1;               // 准备读按键

    while (1) {
        if (KEY == 0) {    // 检测按下
            delay();       // 消抖
            if (KEY == 0) {
                LED = !LED;
                while (KEY == 0);  // 等待释放
            }
        }
    }
}
```

#### 功能与知识点

- 功能：按键每按下一次，LED 状态翻转一次。
- 体现知识点：I/O 读写、按键消抖、等待释放、位操作。
- 编程思路：检测按下 -> 延时消抖 -> 再次确认 -> 执行动作 -> 等待释放。

## 第9章强化：系统扩展

### 必背知识点

- **系统扩展必画三类线：地址线、数据线、控制线。**
- **P0：低 8 位地址/数据复用；P2：高 8 位地址。**
- **ALE 控制 74LS373 锁存低 8 位地址。**
- **程序存储器用 `PSEN` 读；片外数据存储器用 `RD/WR`。**
- **4KB 存储器需要 12 根地址线。**
- **地址范围 = 容量 + 片选逻辑共同决定。**

### 考试常考问法

- “补全 8051 外扩 EPROM/SRAM 的连接线。”
- “给出某芯片容量和片选信号，求地址范围。”
- “为什么要用 74LS373？”
- “把程序存储器最低地址内容传到数据存储器最高地址单元。”
- “确定外扩 I/O 口地址并编程访问。”

### 易错点辨析

| 易错点 | 辨析 |
|---|---|
| 只按芯片容量算地址范围 | 不够。还要看片选信号由哪些高位地址决定。 |
| 4KB 需要 13 根地址线 | 错。4KB = 2^12，需要 A0～A11 共 12 根。 |
| 程序存储器和数据存储器控制线一样 | 错。程序存储器常用 `PSEN`，数据存储器用 `RD/WR`。 |
| P0 可直接接存储器 A0～A7 并保持稳定 | 错。要通过 74LS373 锁存。 |

### 系统扩展速背表

| 信号/芯片 | 作用 |
|---|---|
| P0 | 低 8 位地址 + 8 位数据复用 |
| P2 | 高 8 位地址 |
| ALE | 地址锁存允许 |
| 74LS373 | 锁存低 8 位地址 |
| `PSEN` | 读外部程序存储器 |
| `RD` | 读外部数据存储器/I/O |
| `WR` | 写外部数据存储器/I/O |
| `EA` | 选择片内/片外程序存储器访问方式，具体接法按题图 |

### 典型程序案例1：把程序存储器一个字节送到片外数据存储器

模拟试卷要求“将程序存储器 2732 最低地址单元内容传送到数据存储器 6232 最高地址单元”。具体地址范围由题图片选决定，文字提取无法完整确认；下面写法是模板，考试时把 `CODE_BASE` 和 `XDATA_TOP` 换成按题图算出的实际地址。

#### 汇编写法

```asm
CODE_BASE EQU 0000H       ; 示例：程序存储器最低地址，实际按题图替换
XDATA_TOP EQU 0FFFH       ; 示例：数据存储器最高地址，实际按题图替换

        ORG 0000H
START:  CLR A             ; A=0，用于 MOVC A,@A+DPTR 取最低地址内容
        MOV DPTR,#CODE_BASE
        MOVC A,@A+DPTR    ; 从程序存储器读取一个字节

        MOV DPTR,#XDATA_TOP
        MOVX @DPTR,A      ; 写入片外数据存储器
        SJMP $
        END
```

#### C51写法

```c
#include <reg51.h>

#define CODE_BASE 0x0000  // 示例地址，考试时按题图替换
#define XDATA_TOP 0x0FFF  // 示例地址，考试时按题图替换

void main(void)
{
    unsigned char code *pcode = (unsigned char code *)CODE_BASE;
    unsigned char xdata *pxdata = (unsigned char xdata *)XDATA_TOP;

    *pxdata = *pcode;      // 程序存储器 -> 片外数据存储器

    while (1);
}
```

#### 功能与知识点

- 功能：从外部/程序存储器指定地址读 1 字节，写入片外数据存储器指定地址。
- 体现知识点：`MOVC`、`MOVX`、DPTR、程序/数据存储空间、系统扩展地址范围。
- 编程思路：先根据题图算地址 -> 用 `MOVC` 读程序空间 -> 用 `MOVX` 写片外数据空间。

### 典型程序案例2：读取外扩输入口并串口发送

该案例对应模拟试卷第二页的 74LS245 外扩输入口题。外设 A 口地址由图中片选决定；下面用 `PORT_A` 作占位。

#### 汇编写法

```asm
PORT_A EQU 8000H          ; 示例外设地址，实际按题图片选计算

        ORG 0000H
MAIN:   MOV SCON,#80H     ; 串口方式2，按题目提示不设 T1 初值

LOOP:   MOV DPTR,#PORT_A
        MOVX A,@DPTR      ; 读取外扩输入口
        MOV SBUF,A        ; 串口发送
WAIT:   JNB TI,WAIT
        CLR TI
        ACALL DELAY20MS
        SJMP LOOP

DELAY20MS:
        ; 题目提示可不写出 20ms 延时子程序
        RET
        END
```

#### C51写法

```c
#include <reg51.h>

#define PORT_A 0x8000     // 示例外设地址，实际按题图片选计算

void delay20ms(void)
{
    // 题目若说明不用写出，可留空
}

void main(void)
{
    unsigned char xdata *port_a = (unsigned char xdata *)PORT_A;

    SCON = 0x80;           // 串口方式2

    while (1) {
        SBUF = *port_a;    // 读外扩输入口并发送
        while (TI == 0);
        TI = 0;
        delay20ms();
    }
}
```

#### 功能与知识点

- 功能：周期性读取外部输入口，并通过串口方式2发送。
- 体现知识点：外扩 I/O 地址、`MOVX`、串口方式2、`SBUF/TI`、延时。
- 编程思路：先由硬件图算端口地址 -> 初始化串口 -> `MOVX` 读外设 -> 写 `SBUF` -> 等待发送完成。

## 考前最后一页：高频辨析总表

| 对比项 | 必须分清 |
|---|---|
| `MOV` / `MOVC` / `MOVX` | 片内数据 / 程序存储器 / 片外数据 |
| `RET` / `RETI` | 子程序返回 / 中断返回 |
| `TR0` / `TF0` | 启动 T0 / T0 溢出标志 |
| `TI` / `RI` | 串口发送完成 / 串口接收完成 |
| P0 / P2 | 低地址+数据复用 / 高地址 |
| 定时器 / 计数器 | 内部机器周期计数 / 外部事件计数 |
| 静态显示 / 动态显示 | 一直驱动 / 轮流扫描 |
| 读引脚 / 读锁存器 | 读外部实际电平 / 读端口锁存状态 |
| 程序存储器 / 数据存储器 | `MOVC/PSEN` / `MOVX/RD/WR` |

## 考前程序设计模板

1. **端口类程序**：定义端口 -> 主循环读/写 -> 必要时延时或消抖。
2. **中断类程序**：写入口地址/中断函数 -> 初始化允许位 -> ISR 中保护现场 -> 短小处理 -> `RETI`。
3. **定时类程序**：确定晶振 -> 算初值 -> 设置 `TMOD` -> 装初值 -> 开中断/启动 -> 中断中重装。
4. **串口类程序**：设置 `SCON` -> 写 `SBUF` -> 等 `TI` -> 清 `TI`。
5. **外扩类程序**：先算地址 -> 区分程序/数据/I/O 空间 -> `MOVC` 或 `MOVX` 访问。
