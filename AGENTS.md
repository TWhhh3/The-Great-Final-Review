# AGENTS.md

本文件是本项目的长期规则文件。后续在本项目中工作时，优先遵守这里的约定。

## 项目目标

本项目用于复习《微机原理》期末考试。`materials` 文件夹中会放入 PPT、PDF、Word、图片等原始复习资料，包括复习重点。

需要根据 `materials` 中的资料生成：

1. 核心复习提纲
2. 重点概念详解
3. 可打印的 PDF 文件
4. 题库 JSON 文件
5. 本地网页自测程序

## 文件夹规则

- `materials`：只放原始资料，不要修改其中内容。
- `extracted`：存放从原始资料中提取出来的文本。
- `output`：存放复习提纲、PDF、题库 JSON 等最终或半最终成果。
- `app`：存放本地网页自测程序。
- `scripts`：存放 Python 脚本。

## 内容要求

- 所有总结、提纲、解释和题库尽量基于 `materials` 中的资料。
- 不要凭空编造。
- 如果资料中没有明确说明，必须标注“资料未明确说明”。
- 复习提纲要适合期末复习，突出核心知识点、常考点和易混点。
- 题库需要包含以下题型：
  - 选择题
  - 填空题
  - 简答题
  - 程序阅读分析题
  - 系统扩展设计题
  - 程序设计题
- 生成题目时，应尽量能追溯到资料中的依据；无法确认依据时，应标注“资料未明确说明”。

## 技术要求

- 优先使用 Python 处理资料、提取文本、生成 Markdown、JSON 和 PDF。
- 网页程序使用原生 HTML、CSS、JavaScript。
- 代码尽量简单、清晰，方便初学者阅读和修改。
- 不引入复杂框架，除非确实有必要并提前说明原因。
- 对 PDF、Word、PPT、图片等资料，应优先通过脚本提取文本，再基于提取文本整理内容。

## 工作流程要求

每次修改后，需要说明：

1. 改了哪些文件。
2. 如何运行相关脚本或网页程序。
3. 生成结果在哪里。
4. 需要用户检查什么。

## 资料使用边界

- 不修改 `materials` 中的原始资料。
- 如果需要清洗、合并、摘录资料内容，应输出到 `extracted` 或 `output`。
- 如果某个结论不是来自资料，而是基于常识或推断，必须明确说明。
- 对不确定内容保持保守，不把推测写成确定结论。

## 网页自测程序要求

- 放在 `app` 文件夹中。
- 使用原生 HTML、CSS、JavaScript。
- 能读取或内置题库 JSON。
- 界面简单清楚，适合本地复习自测。
- 功能优先级：
  1. 题目浏览
  2. 按题型筛选
  3. 答题与查看答案
  4. 简单统计正确率
